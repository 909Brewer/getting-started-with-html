The Nitty Gritty of the Web
There are four main components commonly used to describe the "web". The most common point of access for an individual is their personal computer with a web browser(1),such as Chrome or Firefox. Most users will communicate their request through the Internet(2), which is the largest computer network, via an Internet protocol. The most common Internet protocol is HTTP(3), which is a protocol that allows the computer to speak with Servers(4). Servers are in essence computers which have been optimized to store and host large amounts of data which are regularly accessed.

Understanding The Web breaks it down fairly well in an easy to understand no nonsense explanation.

HTML
Hyper Text Markup Language makes up the bulk of online documents. Anything can exist on the web, however HTML will be the most prolific type of document you will find.
What HTML is made of:
1.-- Text content (what you see on the page)
2.-- Markup (what the content "looks" like)
3.-- References to other documents (images, videos)
4.-- Links to other pages


HTML Markup and Elements
The whole markup; opening tag, content, and closing tag, is referred to as an element. A body containing "nested" tags, or multiple tags inside of each other, is perfectly acceptable. A very simple and generic expample would be:
< name> contents < /name>
opening tag         closing tag

Some common examples of tags would be:
    < b > - bold
    < em> - emphasis

HTML ATTRIBUTES

Attributes provide additional information about HTML elements such as defining an image, or establishing a link. This is typically written as a tag, then attribute, value of the attribute, "contents", then the closing tag as seen below.
< tag attr="value"> contents < /tag>

A common example of a "tag" would be the Anchor (which establishes a link) - < a>
< a    href=  "www.reddit.com">Shamlessplug< /a>
Anchor     Attribute     Value     Contents      Closing anchor

A second example is the Image tag ------- < img>
< img   src  =    "url"   alt="text">
Tag    Source   URL    alt (which stands for alternate) "text displayed if image doesn't load"

Since images don't have any contents they are considered a "void tag", and thus don't have a closing tag.

Whitespace

An interesting fact about HTML is that the computer does not recognize "space" in it's traditional role. No matter how many "spaces" an individual may type into a line, the computer only codes it as 1 gap until the next character. Some common ways to reflect spacing, or dropping down to a new line include the break, and paragraph tags. There are essentially two forms of formatting, inline, and block. The easiest way I found to remember this is that The block examples are structural elements, while the inline elements are text-based.

< br> - break gives you whitespace so lines don't run on together, since the break tag has no content, it is a void tag. It is considered an inline tag as it is text based and not structural.
< p> - paragraph is for block style of creating white space, and you can adjust height and width etc. Since there is content to block style, there are closing tags.
